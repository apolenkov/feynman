Контракт TargetAdapter

bin/feynman.ts (или lib/target-adapter.ts если разрастётся) 
interface TargetAdapter {
  name: 'claude' | 'codex' | 'opencode';
  paths: {
    home: string;     // ~/.claude | ~/.codex | ~/.config/opencode
    state: string;    // <home>/.feynman/state.json
    flag: string;     // <home>/.feynman-active
    settings: string; // settings.json | hooks.json | opencode.json
  };
  install(opts: InstallOpts): Promise<void>;
  uninstall(opts: UninstallOpts): Promise<void>;
  doctor(): Promise<DoctorResult>;
}

const adapters: Record<TargetName, TargetAdapter> = {
  claude:   createClaudeAdapter(),
  codex:    createCodexAdapter(),
  opencode: createOpenCodeAdapter(),
};

05

Open questions (с defaults)

Q1 — OpenCode понимает абсолютные пути в instructions: [...]?

Default: пишу absolute. Если в Phase B.3 эмпирически не подхватилось — fallback на relative от ~/.config/opencode/. Это единственный вопрос, который теоретически может всплыть в чат.

Эскалация: только если оба варианта не работают

Q2 — opencode.json иногда содержит JSONC (комментарии)?

Default: сначала JSON.parse. На fail — минимальный regex strip-comments без npm-deps. Решается автономно при первом столкновении.

Эскалация: не требуется

Q3 — Есть ли OPENCODE_CONFIG_DIR env var?

Default: по разведке такой переменной нет — использую жёсткий ~/.config/opencode. Если позже появится — добавлю env probe по аналогии с CLAUDE_CONFIG_DIR/CODEX_HOME.

Эскалация: не требуется

Q4 — lib/target-adapter.ts отдельным файлом или inline в bin/?

Default: inline пока fits в ~150 LoC; вынести в lib/ если перевалит за 200. Решается автономно по факту в Phase A.

Эскалация: не требуется
