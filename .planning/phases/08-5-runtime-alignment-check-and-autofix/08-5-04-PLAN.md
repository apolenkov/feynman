# Plan 08-5-04 — CLI integration + Stop-hook + docs

## Goal

Wire the autofix engine end-to-end: `feynman-lint --fix <file>` writes corrected
text in-place; the Stop-hook variant applies autofix to model output before the
user sees it; falls back to rule-feedback if no frame nodes detected. Update
public docs.

### Task 3 — GREEN: Implement CLI flag
In `bin/feynman-lint.js`:

```js
const argv = process.argv.slice(2);
const fixIdx = argv.indexOf('--fix');
const fixMode = fixIdx !== -1;
const files = fixMode ? argv.slice(0, fixIdx).concat(argv.slice(fixIdx + 1)) : argv;

if (fixMode) {
  const { autofix } = require('../lib/lint/autofix');
  for (const file of files) {
    const before = fs.readFileSync(file, 'utf8');
    const after = autofix(before);
    if (before !== after) fs.writeFileSync(file, after);
  }
  process.exit(0);
}
// existing detection-only path unchanged
```

Run CLI tests — pass.

### Task 4 — GREEN: Implement Stop-hook autofix
In `hooks/feynman-lint.js`:

```js
const { parse } = require('../lib/lint/parser');
const { autofix } = require('../lib/lint/autofix');

// inside the handler, after reading model response:
const ast = parse(modelResponse);
const frames = ast.filter(n => n.kind === 'frame');
if (frames.length > 0) {
  const fixed = autofix(modelResponse);
  if (fixed !== modelResponse) {
    return jsonOut({ hookSpecificOutput: { additionalContext: '<feynman-autofix>\n' + fixed + '\n</feynman-autofix>' } });
  }
}
// fallback: existing rule-feedback path (unchanged)
```

Run hook tests — pass.
