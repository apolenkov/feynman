<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| plugin.json → Claude Code plugin loader | Claude Code reads plugin.json to register the plugin. Malformed JSON or invalid version causes load failure. |
| README.md settings.json example → user's ~/.claude/settings.json | User copies the settings.json block from README into their own settings file. The example path is a placeholder — user substitutes their own absolute path. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | plugin.json (malformed version string) | mitigate | Version must be valid semver `MAJOR.MINOR.PATCH`. Validated by node script in task verify block. Claude Code rejects plugins with invalid versions at load time. |
| T-03-02 | Spoofing | plugin.json hooks key (bug #10225) | mitigate | plugin.json must NOT contain a `hooks` key. If hooks were declared here, they would silently never fire, creating a false sense of security. Validated by `if(m.hooks) process.exit(1)` in verify script. |
| T-03-03 | Tampering | README settings.json example (path injection via copy-paste) | accept | The example uses a placeholder path `"/absolute/path/to/feynman/..."`. Users substitute their own path. Since they control their own ~/.claude/settings.json and the path runs `node` (not `eval`), no injection surface beyond what the user already controls. ASVS L1 does not require mitigations here. |
</threat_model>
