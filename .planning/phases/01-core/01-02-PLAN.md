Create directory `hooks/` and write `hooks/feynman-activate.js` as a zero-dependency CJS Node.js script.

**Required module header:**
```javascript
#!/usr/bin/env node
// feynman — UserPromptSubmit hook — injects ASCII diagram rules on every prompt
// Zero dependencies. CJS only (no ESM — matches caveman pattern).
'use strict';
const fs = require('fs');
const path = require('path');
const os = require('os');
```

**Path constants block (exact names — Phase 2 skills will expect these same paths in state.json):**
```javascript
const HOME = os.homedir();
const FEYNMAN_DIR = path.join(HOME, '.claude', '.feynman');
const STATE_PATH = path.join(FEYNMAN_DIR, 'state.json');
const FLAG_PATH  = path.join(HOME, '.claude', '.feynman-active');
const RULES_PATH = path.join(__dirname, '..', 'rules', 'feynman-activate.md');
const DEFAULT_STATE = { enabled: true, intensity: 'full', count: 0 };
```

**Main execution flow (stdin → JSON parse → guard → bootstrap → select → output):**

Implement as stdin accumulator with process.stdin.on('end') callback — exact caveman pattern. Wrap everything inside the 'end' callback in a try/catch that calls `process.exit(0)` on any error (silent fail, per D-07).

**Step 1 — stdin parse:** `const data = JSON.parse(input);`

**Step 2 — session_id path-traversal guard (security — VERIFIED pattern from gsd-context-monitor.js):**
```javascript
const sessionId = data.session_id || '';
if (sessionId && /[/\\]|\.\./.test(sessionId)) process.exit(0);
```

**Step 3 — flag file check (bug #35713 workaround — disabled plugin still injects):**
```javascript
if (!fs.existsSync(FLAG_PATH)) process.exit(0);
```

**Step 4 — state.json read with first-run bootstrap (D-05):**
```javascript
let state;
try {
  state = JSON.parse(fs.readFileSync(STATE_PATH, 'utf8'));
} catch (e) {
  if (e.code !== 'ENOENT') process.exit(0); // corrupt — fail safe per D-07
  // First run: bootstrap
  fs.mkdirSync(FEYNMAN_DIR, { recursive: true });
  state = Object.assign({}, DEFAULT_STATE);
  fs.writeFileSync(STATE_PATH, JSON.stringify(state, null, 2));
  fs.writeFileSync(FLAG_PATH, state.intensity); // activate feynman from first run
}
```

**Step 5 — enabled check:**
```javascript
if (!state.enabled) process.exit(0);
```

**Step 6 — rules file read + variant extraction (D-01, D-02, HOOK-04):**
```javascript
let rulesText;
try {
  const rulesContent = fs.readFileSync(RULES_PATH, 'utf8');
  const intensity = (state.intensity === 'lite' || state.intensity === 'ultra') ? state.intensity : 'full';
  const m1 = '<!-- ' + intensity + ' -->';
  const m2 = '<!-- /' + intensity + ' -->';
  const i1 = rulesContent.indexOf(m1);
  const i2 = rulesContent.indexOf(m2, i1);
  if (i1 === -1 || i2 === -1) process.exit(0);
  rulesText = rulesContent.slice(i1 + m1.length, i2).trim();
} catch (e) {
  process.exit(0); // missing rules file — self-heal silently
}
if (!rulesText) process.exit(0);
```

**Step 7 — increment counter + write state (HOOK-05):**
```javascript
state.count = (state.count || 0) + 1;
try { fs.writeFileSync(STATE_PATH, JSON.stringify(state, null, 2)); } catch (e) { /* non-fatal */ }
```

**Step 8 — stdout output (HOOK-01 — bug #13912 workaround):**
```javascript
process.stdout.write(JSON.stringify({
  hookSpecificOutput: {
    hookEventName: 'UserPromptSubmit',
    additionalContext: rulesText
  }
}));
```

**CRITICAL — no other stdout writes anywhere in the file.** `console.log` is forbidden. Any non-JSON stdout causes a red error banner in Claude Code (bug #13912).

**Outer catch block:**
```javascript
} catch (e) {
  process.exit(0); // silent fail — never show hook errors in UI
}
```

**No self-invoking logic outside the stdin 'end' callback.** The entire logic lives inside the callback.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Claude Code runtime → hook stdin | Claude Code writes JSON to hook's stdin. The `session_id` field comes from Claude Code and is used in a file path — must be sanitized. |
| hook → ~/.claude/.feynman/state.json | Hook reads and writes state.json in the user's home directory. Corrupt or attacker-controlled state could affect behavior. |
| hook → ~/.claude/.feynman-active | Hook reads this flag file to determine enabled state. An attacker who can write to ~/.claude/ could enable/disable feynman. |
| hook stdout → Claude Code | Hook writes JSON to stdout. Claude Code injects the additionalContext value into Claude's context. Content becomes part of Claude's system context. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | session_id → /tmp/feynman-{id}.injected path | mitigate | Reject session_id values containing `/`, `\`, or `..` before any path construction: `if (/[/\\]|\.\./.test(sessionId)) process.exit(0)`. Verified pattern from gsd-context-monitor.js. |
| T-02-02 | Tampering | state.json (corrupt or crafted values) | mitigate | JSON.parse failure on corrupt state → exit 0 silently (D-07). Intensity value validated against allowlist `['lite','full','ultra']` before use in marker construction — any other value falls back to `'full'`. |
| T-02-03 | Tampering | FLAG_PATH symlink attack | accept | Hook only reads (existsSync) the flag file — never writes in the normal path. Write occurs only on first-run bootstrap. Symlink could redirect write to an arbitrary file, but: (a) the write is just an intensity string, not executable code; (b) first-run only fires once; (c) ASVS L1 does not require O_NOFOLLOW for non-privileged writes. Risk accepted at L1. |
| T-02-04 | Tampering | RULES_PATH traversal via __dirname | accept | `__dirname` is resolved by Node.js from the hook file's actual location — not from user input. The path is static and cannot be influenced at runtime. No traversal risk. |
| T-02-05 | Spoofing | additionalContext injection via rules content | mitigate | Rules file uses declarative phrasing (D-08) to avoid Claude's injection detector. This is enforced in Plan 01. Hook injects the text verbatim — it does not sanitize content, trusting that Plan 01's grep gates caught imperative phrasing. |
| T-02-06 | Denial of Service | Oversized additionalContext (context bloat) | mitigate | Each variant capped at 8,000 chars (D-09, enforced in Plan 01). Hook injects verbatim after Plan 01's size gates pass. If a user manually edits the rules file to exceed 8,000 chars, injection still works but uses more context window — this is user-controlled. |
| T-02-07 | Information Disclosure | state.json world-readable | accept | state.json stores only: enabled bool, intensity string ('lite'/'full'/'ultra'), count integer. No secrets, no PII. File lives in ~/.claude/.feynman/ which is user-owned. No sensitive information at risk. |
</threat_model>
