```javascript
// From RESEARCH.md Pattern 1 — extractVariant() used by the hook:
function extractVariant(rulesContent, intensity) {
  const marker = `<!-- ${intensity} -->`;
  const endMarker = `<!-- /${intensity} -->`;
  const start = rulesContent.indexOf(marker);
  if (start === -1) return null;
  const end = rulesContent.indexOf(endMarker, start);
  if (end === -1) return null;
  return rulesContent.slice(start + marker.length, end).trim();
}
// intensity is one of: 'lite' | 'full' | 'ultra'
// The hook calls: extractVariant(fileContent, state.intensity)
// Returned text becomes additionalContext injected into every Claude prompt.
```

**LITE variant** (per RULE-01, RULE-02, RULE-03 — flow+trees only):
- Trigger: sequential flows (A then B then C, or numbered steps with arrows) → ASCII boxes+arrows
- Trigger: hierarchical relationships with 3+ levels → ASCII tree with ├── and └──
- Negative conditions (WHEN NOT TO DRAW): single-fact answers, pure code blocks, answers under 4 lines without explicit structure, comparisons without multiple criteria, any response where the user asked for prose only
- ASCII syntax reference: `[Box] --> [Box] --> [Box]` for flows; `parent\n├── child\n└── child` for trees
- Phrasing: declarative facts ONLY — "Responses that contain sequential flows include an ASCII diagram." NOT "Draw a diagram for flows."

**FULL variant** (per RULE-01, RULE-02, RULE-03 — all five types, default):
Must include everything from lite PLUS:
- Trigger: comparisons of 2+ options across multiple criteria → side-by-side ASCII columns (max 3 cols, ≤10 words per cell)
- Trigger: status summaries longer than 5 lines → ┌─ title ─┐ ... └─┘ frame blocks
- Trigger: priority orderings of 3+ items → ▲ (high) ... ▼ (low) vertical scale
- Same negative conditions as lite, plus: where a table or list already captures the structure clearly enough without a diagram, no additional ASCII diagram is added
- Phrasing: declarative facts throughout

**ULTRA variant** (per RULE-03 — force diagrams even for short responses):
Must include everything from full PLUS:
- Any response with 2+ items in a list → at minimum a simple ASCII tree or column layout
- Any response with any structure (even short) → includes an ASCII diagram
- Only exception: a single sentence of pure prose with no enumerable items
- Phrasing: declarative facts throughout

**D-08 compliance check before writing:** Read every rule sentence. If it starts with "Always", "Never", "You must", "Make sure to", "Draw" as a command — rephrase. Valid openers: "Responses that...", "When a response contains...", "Answers that include...".

**D-09 size check after writing:** For each extracted section (content between markers), character count must be < 8,000. Run mentally: lite section should be ~500-900 chars, full ~800-1400 chars, ultra ~900-1600 chars. If any section approaches 8,000, trim by shortening the ASCII syntax examples, not by removing trigger conditions.
