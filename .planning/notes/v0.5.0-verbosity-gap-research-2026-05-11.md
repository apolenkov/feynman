# Verbosity gap research

## Problem

```
arm                  total chars   Δ vs baseline   Δ vs v0.3.x
v0.2.x baseline      6653          —               −23%
v0.3.x current       8683          +31%            —
v0.3.x + ladder      8375          +26%            −3.5%
```

v0.3.x rules are **57% smaller** in bytes than v0.2.x (4480 vs 10450) yet model output is **31% longer**. The smallest-visual-first ladder (v0.4.0) only closes 3.5%. The remaining 27% gap needs a different intervention.

## What the ladder did NOT do

14 of 15 prompts were already at the minimum-viable visual on v0.3.x. The ladder had no lighter form to drop them to:

- sequence → inline arrow flow (already minimum)
- hierarchy → ASCII tree (already minimum)
- comparison → markdown table (already minimum)
- branching / state-machine / mapping → ASCII diagrams (already minimum)
- `none` class → prose (suppress works correctly)

Only `status-small` (4 items) benefited: frame → dot-leader, −49% on that one prompt. That's the L11 case made redundant at generation time. Good but small.

## Hypothesis space (untested)

The +31% is NOT primarily about over-heavy visuals. Likely sources:

### H1: more elaborate captions and inline labels

v0.2.x: `[client]-[api]-[server]`
v0.3.x: `[Client] → [API Gateway] → [Service]`

The XML rule structure suggests "draw it well" → model adds descriptive labels.

**Test:** sample 5 prompts where both arms produce a sequence-arrow flow. Compare average label length per box. Hypothesis confirmed if v0.3.x has ≥20% longer mean.

### H2: classify-first chain-of-thought preamble

v0.3.0 introduced explicit `<contract>` clause: "Classify shape: sequence, hierarchy, …, or none". This is good for accuracy but may cause the model to narrate its reasoning before drawing.

**Test:** count tokens BEFORE the first diagram char (`┌`, `├`, `→`, `|`) in each arm's response. Hypothesis confirmed if v0.3.x has ≥25% more pre-diagram tokens.

### H3: prose around the diagram

v0.3.x rules emphasize "structure → visual" — but the model may add explanatory prose AROUND the visual ("Here's the architecture as a tree:", "Below is the comparison table:"). v0.2.x older format was less prescriptive, may have produced barer output.

**Test:** strip diagram blocks from responses; compare remaining-prose chars. Hypothesis confirmed if v0.3.x has ≥20% more prose.

### H4: amplify-vs-channel ambiguity

v0.3.0 introduced `<patterns selection="one-of">` for SDLC patterns (retro, handoff, etc.) and "amplify" semantics for structure that exists but baseline didn't produce. The model may be over-amplifying — drawing diagrams in situations where v0.2.x would have stayed prose.

**Test:** count diagram-producing responses across arms. v0.3.x should have more `response_has_diagram: true` if amplify is over-triggering.

Looking at Phase 11 data: v0.2.x produced 12/15 diagrams, v0.3.x produced 11/15. Roughly equal. **H4 likely refuted** by existing data.

## Candidate interventions (each falsifiable)

### A. Caption-brevity rule

Add to `<contract>` of each intensity:

```
Captions/box labels: shortest noun phrase that disambiguates. No articles, no
verbs in labels. "client → api → db" not "the Client → the API Gateway → the
Service Database".
```

**Byte cost:** ~150 bytes per intensity. **Budget impact:** 4443 → 4893 — over budget; need ~450 bytes freed.

**Expected impact:** -10% to -20% on response chars, hits H1.

### B. No-narration-before-visual rule

```
Classify shape silently. When drawing, the diagram comes first; explanation
after, only if the diagram alone is ambiguous. No "Here is the X:" preamble.
```

**Byte cost:** ~120 bytes per intensity. **Expected impact:** -8% to -15%, hits H2.

### C. Response-length budget per class

```
Total response: prose < 50 words for structural prompts; <120 for general.
The visual carries the structure; prose is supplement, not duplication.
```

**Byte cost:** ~100 bytes per intensity. **Expected impact:** -15% to -25% across the board, hits H3.

### Combined ABC

If all three land, predicted total: −30% to −50% from v0.3.x baseline (which would put v0.4.x+ at roughly v0.2.x size with v0.3.0 compliance gains preserved).

Budget: ~370 bytes needed across all 3 contracts. Current slack: 37 bytes. Need to free 333 bytes via compaction — possible candidates:

- Drop `<examples>` block in `full` intensity (saves ~250 bytes) — could move to `docs/visual-patterns.md` and reference it
- Trim some less-used SDLC patterns from the `<patterns>` mutex list (saves ~80-150 bytes)
- Inline syntax in trigger-table notes column (already done)

## Measurement plan for v0.5.0

1. **Re-run harness with 50 prompts** (current corpus = 15, too noisy for −5% claims). Original Phase 11 spec was 50.
2. **3 baseline arms:** v0.2.x | v0.3.x | v0.3.x+ladder (already have).
3. **3 new candidate arms:** v0.4.x+A (caption brevity) | +B (no narration) | +ABC (combined).
4. **Cost in subagent runs:** 6 arms × 50 prompts = 300 responses to lint. ~30-60 min wall time across parallel subagent batches.
5. **REPORT.md:** which intervention closes the most of the 31% gap; trade-off vs. lint compliance.

## When to do this

After v0.4.0 has 2-4 weeks in the wild — real session logs will provide
external evidence for which hypothesis matters most. Premature optimization
would burn the harness budget on noise.

## What NOT to do

- Don't add ALL of A+B+C without measuring — additive rules can over-correct
  (model becomes terse and loses helpful explanation in places where structure
  alone is too cryptic).
- Don't expand the corpus beyond 50 prompts in v0.5.0 first run — diminishing
  returns vs. cost.
- Don't propagate the 31% verbosity claim into release notes as a known bug.
  It is a measurement on a 15-prompt corpus, not a customer-reported issue.

## Open questions

1. Is v0.3.x verbosity actually a problem? Phase 11 measured it but never asked
   "did users complain about response length?" If users like the explanations,
   "more chars" may not be a regression.
2. Does response-length budget interact with `output_style` axis? `short` already
   suppresses heavy visuals; ABC interventions would compound. Worth A/B on the
   compound preset.
3. Where exactly does the 31% live in the response structure? Histogram of
   chars-before-first-diagram-char would isolate H2 from H1+H3.
